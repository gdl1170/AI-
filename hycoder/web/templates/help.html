🧠 Cos'è AI+?

AI+ è un assistente AI ibrido che combina modelli locali (Ollama) e cloud (OpenAI, OpenRouter, Gemini, opencode) con routing intelligente, agenti con tool, knowledge base RAG, note in stile Obsidian, e un ambiente di sviluppo integrato nel browser.

🔄 Architettura ibrida: Routing + KB + Persistenza

AI+ cresce con la tua Knowledge Base. Ogni documento, file o URL che importi arricchisce il bagaglio informativo del sistema. Quando fai una domanda in chat, il modello — sia locale che online — viene sempre potenziato con il contesto della Knowledge Base (RAG), per risposte più rapide, precise e pertinenti.

📝 Prompt → 📚 Knowledge Base → 🧠 Router →
🖥 Locale + KB — oppure — ☁ Online + KB
→ 💬 Risposta con fonti

Il router valuta complessità del prompt, keyword tecniche e rilevanza KB per scegliere il modello migliore. In entrambi i casi, i chunk più rilevanti della Knowledge Base vengono iniettati automaticamente nel contesto. Puoi forzare la modalità con 🖥 🔧 ⚡ nella chat. Il risultato finale include sempre le fonti consultate.

📚 Knowledge Base → 💾 Salvataggio automatico su disco ↻ 🔄 Caricamento all'avvio

📚 Knowledge Base — Il cuore del sistema

La Knowledge Base è il cuore pulsante di AI+. Più dati importi — file, directory, URL, repository GitHub — più il sistema diventa preciso e veloce. Il motore RAG (TF-IDF + embedding) indicizza ogni contenuto in chunk, e il modello locale li usa come contesto per rispondere senza dover chiamare API esterne.

🧩 Chunking intelligente

I file vengono suddivisi in chunk da ~600 caratteri con overlap di 150 caratteri. Supporto multi-formato: PDF, Markdown, Python, JS, HTML, CSS, JSON, YAML, Go, Rust, testo semplice.

Automatico

🔍 Ricerca ibrida TF-IDF + Embedding

La ricerca combina TF-IDF (peso 0.3) per match lessicale e embedding vettoriale (peso 0.7) per similarità semantica. Se l'embedding non è disponibile, usa solo TF-IDF.

Ponderato

💾 Persistenza automatica

L'indice (TF-IDF + vettori) viene salvato su disco in _index.json.gz dopo ogni modifica. I dati sopravvivono a riavvii del server. All'avvio, l'indice viene caricato automaticamente.

Durevole

⚡ RAG gating intelligente

Il sistema non interroga la KB per query banali (saluti, ringraziamenti) usando una lista di pattern predefiniti. Per query con keyword tecniche, forza sempre la ricerca RAG.

Ottimizzato

📊 Grafo della conoscenza

Visualizzazione interattiva dei chunk come grafo (forza, albero, radiale). I nodi sono raggruppati per tipo file, le connessioni rappresentano similarità semantica.

Visivo

📝 Wiki pagine

Per ogni chunk viene generata automaticamente una pagina wiki con metadati (fonte, tipo, dimensione). Il pulsante "Suggerisci" propone pagine correlate.

Auto-generato

🗑 Reset e pulizia

La KB può essere svuotata con un clic ("Svuota KB"). Le fonti possono essere rimosse singolarmente. Il reset cancella anche l'indice su disco.

Controllato

🌐 Importazione multi-formato

Importa file singoli (drag & drop), directory (browser file con pattern), URL (download e indicizzazione HTML). Supporto pattern glob: *.py, *.md, *.

Universale

⚡ Skills — Adatta skill da altre piattaforme

Le Skills ti permettono di importare assistenti specializzati da altre piattaforme (opencode, ChatGPT GPT, Claude Project, custom) e usarli in AI+. Il sistema rileva automaticamente la piattaforma di origine e adatta il formato.

📥 Import automatico

Carica un file .md, .json o .txt e il sistema rileva la piattaforma dal nome file + struttura JSON (presenza di campi come instructions, system_prompt, categories).

Drag & drop

🔄 Adattamento automatico

La skill viene convertita in formato unificato AI+ con campi: nome, descrizione, prompt, piattaforma di origine, modello preferito, temperatura.

Cross-platform

📤 Esportazione

Ogni skill può essere esportata in formato JSON per backup o condivisione. Il CLI bin/adapt-skill permette di importare/esportare da terminale.

Portabile

📋 Gestione libreria

Elenco completo delle skill importate con dettaglio modal, copia prompt, eliminazione. Le skill persistono su disco in ~/.config/hybrid-coder/skills/.

Persistente

🤖 Agenti e Tool

Gli agenti sono assistenti specializzati con istruzioni di sistema, modello preferito e permessi tool. Seleziona un agente dalla barra della chat per attivarlo.

📖

read

Legge file e directory. Supporta offset/limit per file grandi.

✏️

write

Crea o sovrascrive file. Crea automaticamente le directory necessarie.

🔍

edit

Modifica file trovando e sostituendo testo esatto (oldString → newString).

🗑

delete

Elimina file o directory. Operazione reversibile solo se versionato.

▶️

run

Esegue comandi shell in sandbox. Comandi consentiti: python, node, git, grep, curl, pip e altri.

🔎

search

Cerca pattern regex nei file del workspace (grep).

🌐

web

Cerca informazioni online (DuckDuckGo) o scarica pagine web.

🧠

memory

Memoria a breve termine: salva, recupera, elenca e pulisce dati tra iterazioni.

Quando un agente con tool abilitati è attivo, la chat mostra ogni tool call con 🔧 read e il risultato con ✅ output.

🔧 Da CLI

AI+ funziona anche da terminale:

  • hy chat — chat interattiva
  • hy web — avvia il server web
  • hy generate "prompt" — output JSON
  • hy learn from-dir <path> — indicizza documenti
  • hy notes list — gestione note
  • hy project create <nome> — scaffold progetto
